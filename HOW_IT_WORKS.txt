HOW IT WORKS
============

ARCHITECTURE
------------
Browser (localhost:5173) --> Spring Boot API (localhost:8080) --> PostgreSQL DB


PARTS
-----
1. PostgreSQL   - stores questions and quizzes
2. Spring Boot  - Java backend, handles all logic, exposes REST API
3. React        - frontend, talks to the API, shows UI in browser


USER FLOW
---------
1. Add a question     --> saved to PostgreSQL
2. Create a quiz      --> backend picks random questions from DB
3. Take the quiz      --> frontend shows the questions
4. Submit answers     --> backend checks answers and returns score
5. See result         --> score shown on screen


HOW TO RUN
----------
Step 1 - Start PostgreSQL (only once)
  brew services start postgresql

Step 2 - Start the backend (terminal 1)
  cd /Users/aghar/Documents/quiz-spring
  mvn spring-boot:run

Step 3 - Start the frontend (terminal 2)
  cd /Users/aghar/Documents/quiz-spring/frontend
  npm run dev

Step 4 - Open in browser
  http://localhost:5173


Q&A
---

Q: What do these commands actually do?

  brew services start postgresql
    - "brew" is Homebrew, a Mac package manager
    - "services start" tells it to run a background service
    - "postgresql" is the database we want to start
    - Result: PostgreSQL database starts running silently in the background

  cd /Users/aghar/Documents/quiz-spring
    - "cd" means "change directory"
    - moves your terminal into the project folder

  mvn spring-boot:run
    - "mvn" is Maven, a Java build tool
    - "spring-boot:run" tells it to compile and start the Java backend
    - Result: the API server starts on http://localhost:8080

  cd /Users/aghar/Documents/quiz-spring/frontend
    - moves your terminal into the frontend folder

  npm run dev
    - "npm" is Node Package Manager, used for JavaScript projects
    - "run dev" starts the frontend in development mode
    - Result: the React app starts on http://localhost:5173

  http://localhost:5173
    - "localhost" means your own computer (not the internet)
    - "5173" is the port number (like a door number) the frontend listens on
    - Opening this in Chrome shows you the app UI
